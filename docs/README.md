## Documentation Standards

### Writing Guidelines

1. **Clear and Concise**: Use simple language and short sentences
2. **Examples**: Include code examples where applicable
3. **Up-to-date**: Keep documentation in sync with code changes
4. **Cross-references**: Link to related documents

### Formatting

- Use Markdown for all documentation
- Include table of contents for long documents
- Use code blocks with language specification
- Add diagrams using Mermaid where helpful

## Contributing to Documentation

1. Follow the existing structure
2. Update the table of contents in this file
3. Use clear, concise language
4. Test code examples before submitting
